## ðŸŽ¯ Overview

- **Profile Management**: Members can view and manage their profiles
- **Member Directory**: Browse and connect with other YPO members
- **Onboarding Flow**: Guided onboarding experience for new members
- **Admin Features**: Administrative tools for managing members and content

## ðŸš€ Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/haggaibb/ypo_insider.git
   cd ypo_connect
   ```

2. **Install Flutter dependencies**
   ```bash
   flutter pub get
   ```

3. **Configure Firebase**
   - Create a Firebase project at [Firebase Console](https://console.firebase.google.com/)
   - Enable Authentication (Email/Password)
   - Create a Firestore database
   - Enable Firebase Storage
   - Run `flutterfire configure` to generate `firebase_options.dart`
   - **Note**: `firebase_options.dart` is excluded from version control for security

4. **Set up Firebase configuration**
   - Copy `firebase.json.example` to `firebase.json` (if available)
   - Configure your Firebase project settings
   - **Important**: Never commit `firebase.json` or `firebase_options.dart` to version control

   ```bash
   flutter build web
   ```

2. **Deploy to Firebase**
   ```bash
   firebase deploy --only hosting
   ```

   Or use the deployment script:
   ```bash
   ./deploy.sh
   ```

### Deployment Configuration

The `firebase.json` file (excluded from git) should contain:

```json
{
  "hosting": {
    "public": "build/web",
    "ignore": [
      "firebase.json",
      "**/.*",
      "**/node_modules/**"
    ]
  }
}
```

## ðŸ”’ Security Notes

### Sensitive Files Excluded from Git

The following files are excluded from version control for security:

- `firebase.json` - Firebase project configuration
- `lib/firebase_options.dart` - Firebase API keys and configuration
- `.firebase/` - Firebase CLI cache
- `.firebaserc` - Firebase project aliases
- `firestore.rules` - Firestore security rules (if containing sensitive logic)
- `firestore.indexes.json` - Firestore indexes (if containing sensitive data)

### Best Practices

1. **Never commit sensitive data**: API keys, credentials, or configuration files with secrets
2. **Use environment variables**: For any configuration that varies between environments
3. **Review Firestore rules**: Ensure proper security rules are in place
4. **Regular security audits**: Review dependencies for vulnerabilities
5. **Keep dependencies updated**: Run `flutter pub outdated` regularly

### Setting Up for New Developers

New developers should:

1. Clone the repository
2. Run `flutter pub get`
3. Set up their own Firebase project or request access to the development project
4. Run `flutterfire configure` to generate `firebase_options.dart`
5. Configure `firebase.json` with their Firebase project settings
